# WinterReport

## 研究に関する書籍

### タイトル
はじめてでも安心！Unity5の教科書　2D3Dスマートフォンゲーム入門講座
### 著者
北村愛美

### 出版
SBクリエイティブ株式会社

### 内容
全てで8つのチャプターに分かれており、各チャプターごとに学んだことをまとめる。
#### Chapter1「ゲーム作りの準備」
- unityのインストールから簡単な画面操作の紹介 
#### Chapter2「C#スクリプトの基礎」
- C#の学習(変数、制御文、配列、関数、クラス)  
Vectorクラスの学習
#### Chapter3「オブジェクトの配列と動かし方」
- ルーレットを回す2Dのゲームを作成を行なった。プロジェクトの作成、オブジェクトをシーンへ配置、スクリプトの作成、スクリプトのアタッチというゲームを動かすための全体的な動作を学んだ。  
- Input.GetMouseButtonDown()(マウスが押されたことを読み取る関数で引数が0なら左クリック、1なら右クリック、2なら中央ボタンを検知)でルーレットを回転、ルーレットを回すのにRotate()(画像を回転させる)関数、回ったルーレットを止めるのに減衰係数として0.96を乗算し加え続けることで自然にルーレットが止まることができると学んだ。
#### Chapter4「UIと監督オブジェクト」
- 画面上でスワイプした長さを速度に変えて車のオブジェクトをできるだけゴールに近づける2Dのゲームの作成を行なった。車とゴールへの距離はUを用いて画面上に表示を行なう。  
- 監督スクリプト内ではテキストのUIに車とゴールまでの距離を導出する。表示する距離を少数点第二位までとするためにToString()(書式指定子として整数型をD、固定小数点型をF)の書式指定子を"F2"とした。
- UIを使用にするにあたりUIを更新するためのスクリプト(本書では監督スクリプト)の導入を行なった。  
- スワイプした長さを測るためにInput.GetMouseButtonDown()とInput.GetMouseButtonUp()を用いて各座標をVectorクラスで読み取り距離を導出した。
- UIと効果音をアタッチするのにあたり、コンポーネントの導入を行なった。効果音を鳴らす際にはGetComponent＜AudioSource＞()を用いてコンポーネントへアタッチをする。
#### Chapter5「Prefabと当たり判定」
- 画面上部から落下する矢をプレイヤーが左右に移動して避ける2Dゲームの作成を行なった。
- Prefabの導入。設計図を元に指定した時にオブジェクトを生成するツール。今回は一定のスパンで矢を画面上部のランダムな位置に生成する。
- アンカーポイントで体力ゲージの画像の位置を右上に指定して配置。この体力ゲージはUIとして、プレイヤーが矢に当たると徐々に減少する。
- 体力ゲージを円形に減少させるためにUIオブジェクトのImageに用意されているFill Amount変数をRadial 360にした。他の用意されている変数として、半円形に画像を切り取るRadial 180、90度の扇形に画像を切り取るRadial 90、横方向に画像を切り取るHorizonal、縦方向に画像を切り取るVertialなどがある。
- 当たり判定として矢オブジェクトのスクリプトにプレイヤーオブジェクトと矢オブジェクトの半径の長さを設定→各円の中心座標の和　<　各円の半径の和　となった際に矢がプレイヤーに当たったと判断する。プレイヤーと矢の座標を求めるのにVectorクラスを用いた。
- InputKeyDown()を用いてキーボード操作、またボタンを配置し、ボタンが押された際に同じ関数を呼び出すことでスマートフォンでの操作も可能となった。
#### Chapter6「Physicsアニメーション」
- 雲の足場を登り頂上のゴールを目指す2Dゲームの作成を行なった。
- Physicsの導入。オブジェクトを物理挙動にしたがって動かすことのできるツール。当たり判定にも用いられる。  
今回はプレイヤーオブジェクトにRigidbody2Dと(Circle)Collider2D、雲オブジェクトにRigidbody2Dをアタッチした。プレイヤーと雲の衝突で物理的な挙動をさせるためには雲オブジェクトにもColliderをアタッチする必用があるが、今回雲は固定のオブジェクトであるためColliderはアタッチせず、雲が重力の影響を受けないようにするためにRigidbody2Dの設定項目の「Is Kinematic」にチェックを入れた。
- Colliderを組み合わせてコライダの形状の変え方を学んだ。
- スペースキーでジャンプ、左右のキーで左右へ移動をPhysicsのRidid2Dに用意されているAddForce関数でプレイヤーに力を加えることで実現した。AddForce関数を使って力を加え続けるとプレイヤーどんどん加速度的に加速してしまうためスピード制限をするために変数maxWalkSpeed = 2.0fを用意しプレイヤーがこのスピード以下である場合のみにAddForce関数を実行するようにした。
- プレイヤーが移動した方向を向くようにするためにlocalScal（）(画像を反転させる)関数を用いた。プレイヤーの画像は右向きであるが、変数key（右のキーボードが押された際に１、左のキーボードが押された際に-1を返す）変数を用い、transform.localScale = new Vector(key,1,1)として各キーボード操作によってX軸方向に画像を反転させる。
- Mechanimの導入。各アニメーションの切り替えタイミングを設定することによって自動的にアニメーションを切り替えて再生を行なうツール。  
今回は各キー操作(左右の移動とジャンプ)に対応したアニメーションの再生を行なった。
- Pthsicsでの当たり判定を学んだ。Physicsでの当たり判定にはCollisionモードとTriggerモードがある。Collisionモードはオブジェクト同士が当たった場合の衝突検出だけではなく、跳ね返りなおの衝突応答も実行される。Triggerモードは衝突検出だけを行ない衝突応答は実行されない（衝突したオブジェクト同士はすり抜ける）。  
今回はTriggerモードを使用してプレイヤーとゴールが触れた際にクリアシーンへ遷移するという流れを実現した。
- ゲーム性を高めるためにジャンプ中に何度もジャンプができてしまうというバグを修正した。修正方法としてプレイヤースクリプトにRigid2D.velocity.y()を用いてこのプレイヤーのY軸の速度が０である場合のみにジャンプができるようにした。  
また、プレイヤーが画面の場外(position.y < -10>)に出た際に最初のゲーム画面に遷移するように修正をした。
#### Chapter7「3Dゲームの作り方」
- イガグリを的に当てる3Dゲームの作成を行なった。
- Terrainの導入。山や川などの地形をペイント間隔で簡単に生成することのできるツール。今回は山と木のいんTerrainを使用した。また、Terrainはテクスチャをインポートすることによってペイントすることができ今回はAssetsのImport PackageのEnvironmentから草と岩のテクスチャをインポートした。
- パーティクル表示の導入。イガグリのインスペクタのAddCmponentからEffectsのParticle Systemのパラメータを調節することによってイガグリが的に当たった際のエフェクトを作成した。
- パーティクルの生成パターンを変化させるためにParticle SystemのEmissionの項目を設定する→毎フレーム生成されるパーティクルの数を表すRate、指定した時間に生成するパーティクルの数を表すBursts。
今回はパーティクルを生成し続ける必用が無いためRateを０、BurstsのTimeを0、MinとMaxを50に設定した。
- パーティクルの再生時間を短くするためにエフェクトの際せ時間を表すDurationと、パーティクルの再生時間を表すStart Lifetimeを1(秒)に設定した。  
- パーティクルをスムーズにフェードアウトさせる。その方法として、パーティクルの透明度を徐々に上げるか、サイズを徐々に小さくする方法がある。  
今回はサイズを徐々に小さくする方法を行なった。Particle System項目のSize over Lifetimeにチェックを入れ、Sizeの黒いエリアを選択しParticle System Curvesから減衰カーブを選択する。
- パーティクルの再生タイミングの設定を行なった。的に当たったときに初めてエフェクトを表示するためにPlay on Awakeのチェックを外す。これがチェックされているとパーティクルをアタッチしたオブジェクトがアクティブになると同時にエフェクトが再生されてしまう。また、今回のエフェクトはループ再生を行なわないためLoopingのチェックも外す。
- タップした方向にオブジェクトを飛ばすためにはタップした座標が必用である。タップした座標はInput.mousePosition()関数で取得可能であるが、mousePositionの値はスクリーン座標の値であるため3Dゲームではこのスクリーン座標をScreenPointToRay()関数に渡すことでワールド座標系でのベクトルを取得する。
- Rayクラスについて。Rayとは名前通り光線のことで光線の座標(origin)と光線の方向(direction)をメンバ変数に持っている。このRayの特徴として、コライダがアタッチされたオブジェクトとの当たりを検知できる。つまり光線がオブジェクトによって遮られた場合には、それを検知できる。
#### Chapter8「レベルデザイン」
- これまでの章で学んだことを総復習する章となっている
- 画面上部から落下してくるリンゴをキャッチする3Dゲームの作成を行なった。リンゴを獲得するとポイントが加算され、ポイントが表示される。画面上部からはリンゴの他に爆弾も落ちてくる。爆弾をキャッチしてしまうと減点されてしまう。
- ScreenPintToRay()関数を用いた当たり判定を学んだ。Physics.Raycast()関数を使って光線がステージオブジェクトと当たったがどうかを調べる。
